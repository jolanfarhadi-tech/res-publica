# Res Publica — Brain Governance Rules

*The rules that govern how this directory itself is maintained, consolidated in one place. None of these are new — each is already stated where the relevant discipline was first established; this file exists so they don't have to be found piecemeal.*

1. **Approved ADRs are never rewritten, only amended.** An amendment is appended as a new, clearly-labeled section (`## Amendment (...)`); the original Context/Decision/Alternatives/Consequences/Future Impact sections stay completely intact. If a decision must be reversed rather than clarified, that requires a new ADR explicitly superseding it — see `../DECISIONS.md`, "Reading the amendments."
2. **One canonical location per concept, everywhere else a link.** The index layer (the 10 top-level files) never restates what a detail-layer document already says in full — see `../README.md`.
3. **Nothing in the Brain may contradict an approved Foundation decision.** If a document appears to disagree with an ADR or the Foundation Architecture, the ADR/Foundation Architecture wins, and the disagreement is a defect in the newer document, not a live open question — see `../README.md`.
4. **A detail-layer document must be the real, unabridged, original artifact — not a paraphrase presented as if it were the original.** If the original artifact's full text is not available to migrate verbatim, the correct action is to record that gap explicitly (what's missing, why, and where the best-available summary lives), not to author new substitute text and present it as the migrated original. Authoring substitute text for a missing approved artifact is regeneration, not migration, and is out of scope for a migration pass. See `../RESEARCH/engineering-security-audit.md` and `../PRODUCTS/experience-blueprint.md` for the two places this rule is currently in effect, and `../PROJECT_BRAIN_STATUS.md` for the full accounting.
5. **A Brain migration pass does not produce new architecture, new features, or a redesign of an approved decision.** Its only legitimate outputs are: moving/consolidating already-approved content, adding thin navigational index files that link to already-approved content, fixing broken cross-references, and honestly documenting gaps it cannot close within that scope.
