# Responsibility Dashboard

## Purpose
Aggregates Responsibility Maps into a collective, navigable view of where responsibility concentrates across all captured accounts — the third Innovation. Canonical name: **Responsibility Dashboard** (UI label: "Dashboard").

## Background
Specified in `brain/FOUNDATION/01_HARM_OPERATING_SYSTEM.md` (Innovation 3). **Naming note:** this is a knowledge product, distinct from any product-engineering "Dashboard" UI module that might exist elsewhere in the organization's technical architecture — related in kind (both aggregate, read-only views) but not the same artifact.

## Core Principles
Aggregate and pattern-level only; never an individually-scored or ranked view.

## Definitions
A read-only view aggregating Responsibility Maps to surface recurring responsible actors or systemic issues.

## Framework
- **Input:** the accumulated set of Responsibility Maps.
- **Core Question:** "Where does responsibility concentrate, and what patterns recur?"
- **Output:** the Responsibility Dashboard.

## Workflow
1. Aggregate Responsibility Maps.
2. Identify recurring patterns.
3. Surface patterns warranting deeper documentation to Responsibility Annexes.

## Roles
**Researcher** — performs aggregation and pattern analysis. **Observer** — may review for process integrity.

## Inputs
The accumulated Responsibility Map set.

## Outputs
Aggregate pattern views, feeding Responsibility Annexes.

## Governance
No per-person or per-Organization score, rank, or comparative display; patterns are named descriptively.

## AI Integration
AI may assist in identifying candidate patterns for human review; it does not publish a pattern as confirmed without human sign-off.
