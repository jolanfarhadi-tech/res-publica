# HARM Lifecycle

## Purpose
Describes, in isolation, the 12-stage operating cycle every civic account moves through — the sequence itself, independent of any single stage's internal detail.

## Background
Extracted from `../foundation/01_HARM_OPERATING_SYSTEM.md` §Framework as its own reference document, since the lifecycle is cited by nearly every other methodology document.

## Core Principles
Sequential dependency; no stage is skipped or reordered; each stage's output is the next stage's required input.

## Definitions
See `../foundation/01_HARM_OPERATING_SYSTEM.md` for full definitions of each stage name.

## Framework
```
Citizen Experience → AHIP → Structured Hearing → Reflection → Expert Review →
Mechanism Identification → Responsibility Evidence → Knowledge Products →
Harm Codex → Community Learning → Institutional Learning → Governance Improvement
```
**Resolved:** this 12-stage cycle is canonical. Website material's "8 steps" reference is a simplified public presentation of the same cycle, not a distinct version.

## Workflow
One directional pass per account, start to finish; a single account may inform multiple Harm Codex entries or Institutional Learning cases if its Mechanism recurs elsewhere.

## Roles
See `../foundation/01_HARM_OPERATING_SYSTEM.md` §Roles — not redefined here.

## Inputs
A Citizen Experience account.

## Outputs
A completed record spanning Responsibility Evidence, Knowledge Products, and (where applicable) a Harm Codex entry.

## Governance
Operates under Res Publica's constitutional principles; no stage introduces scoring, ranking, or gamification.

## AI Integration
AI may assist within individual stages (see each stage's own document); it does not alter, skip, or reorder the sequence itself.
