# Responsibility Annexes

## Purpose
Produces a deeper, evidence-backed account of a specific responsibility pattern surfaced in the Responsibility Dashboard — the fourth Innovation. Canonical name: **Responsibility Annexes** (UI label: "Annexes").

## Background
Specified in `brain/FOUNDATION/01_HARM_OPERATING_SYSTEM.md` (Innovation 4).

**Second origination path (architecture decision):** an Annex may also originate directly from a single Structured Hearing's Evidence Package, reviewed and approved by a Scientific Review Committee, rather than from an aggregate Dashboard pattern. Both paths produce the same kind of Annex and receive the same downstream treatment. See `brain/FOUNDATION/03_ANNEX_BLOCKCHAIN_CIVIC_CONTRIBUTION_ARCHITECTURE.md` for the full lifecycle, the Scientific Review Committee's role, and the Blockchain Annex Block produced upon approval.

## Core Principles
Human verification by a reviewer distinct from the drafter; evidence-backed, citable, defensible.

## Definitions
A Responsibility Annex is a documented, evidence-backed account of one responsibility pattern — represented as a specific type of Publication.

## Framework
- **Input:** a pattern or cluster from the Responsibility Dashboard.
- **Core Question:** "What is the full, evidenced account of this specific responsibility pattern?"
- **Output:** a Responsibility Annex.

## Workflow
**Aggregate path** (from a Dashboard pattern):
1. Select a pattern warranting deeper documentation.
2. Conduct research and evidence-gathering.
3. Draft the Annex.
4. A named reviewer, distinct from the drafter, verifies it.
5. Publish as a Knowledge Product, feeding Civic Intelligence and (where relevant) the Harm Codex.

**Direct path** (from a single Structured Hearing, per `brain/FOUNDATION/03_ANNEX_BLOCKCHAIN_CIVIC_CONTRIBUTION_ARCHITECTURE.md`):
1. A Structured Hearing produces an Evidence Package.
2. The Scientific Review Committee reviews it through Scientific Review's four levels (Expert Review, Structured Hearing, Narrative Coding + Normative Alignment, Governance Review Gates — see that document §7 for full detail); approves or returns it for revision.
3. Once approved, the Evidence Package becomes an Annex.
4. A Blockchain Annex Block (integrity/timestamp/approval record only — never raw testimony) is produced.
5. The Annex becomes eligible for Civic Contribution mapping.

## Roles
**Researcher** — drafts (aggregate path). **Reviewer** — verifies (aggregate path). **Expert** — contributes domain review where needed. **Scientific Review Committee** — approves (direct path only; see `brain/FOUNDATION/03_ANNEX_BLOCKCHAIN_CIVIC_CONTRIBUTION_ARCHITECTURE.md`).

## Inputs
A Responsibility Dashboard pattern.

## Outputs
A published Responsibility Annex.

## Governance
Subject to the same human sign-off discipline as any Publication; no Annex publishes without named review.

## AI Integration
AI may assist in drafting or citation-checking; it does not verify or publish an Annex itself.
