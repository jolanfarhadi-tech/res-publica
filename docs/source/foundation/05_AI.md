# AI

## Purpose
States how AI is used, and explicitly bounded, across the entire Res Publica ecosystem.

## Background
Reused from the organization's existing AI Governance Hierarchy and Constitution AI Governance section — not a new AI policy.

## Core Principles
AI assists; AI never replaces humans; AI never validates evidence; AI never determines responsibility; AI never makes governance decisions. Human approval is mandatory at every consequential step.

## Definitions
An **Executive AI Role** coordinates which Operational Agent applies to a concern and escalates to Human Approval Authority — it has no approval authority of its own. An **Operational Agent** performs a bounded, defined function (e.g., flagging a compliance concern) and never acts outside its defined Allowed/Forbidden Actions.

## Framework
```
Executive Layer → Operational Layer → Skills → Plugins → MCP Tools → Execution
```
Execution is always gated by Human Approval Authority — no layer bypasses this.

## Workflow
An AI capability is proposed → mapped to an existing Operational Agent or flagged as requiring new approval → scoped to its Allowed Actions → executed only with the applicable human sign-off.

A task or query within an AI capability's defined scope.

## Outputs
