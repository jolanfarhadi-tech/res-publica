# Terminology

## Purpose
Enforces one definition per concept across the entire documentation tree.

## Background
Assembled from terms confirmed consistent throughout this documentation set.

## Core Principles
One canonical term per concept; no competing synonyms; "Trust" never "Reputation."

## Definitions
See `../glossary/TERMS.md` for the full term list — this document states the *rule*, not the terms themselves.

## Framework
Every document's Definitions section must match `../glossary/TERMS.md` exactly; any drift is a documentation defect, not a valid variation.

## Workflow
Before a new document is written, its terms are checked against `../glossary/TERMS.md`; any new term is added there, not defined ad hoc.

## Roles
Not specified — applies to all contributors.

## Inputs
Any draft document.

## Outputs
Terminology-consistent documentation.

## Governance
Terminology changes follow `../foundation/04_GOVERNANCE.md`.

## AI Integration
Not applicable.

## Examples
"Trust" (never "Reputation"); "Responsibility Evidence" (never "Contribution Credit" — see `brain/GOVERNANCE/CONTRIBUTION_CLARIFICATION.md` for the full reasoning).
