# Writing Style

## Purpose
States how this documentation is written — register, tone, and language rules.

## Background
Confirmed from project material's own explicit writing conventions.

## Core Principles
Official documentation register, not essay or marketing; precise statements; internally consistent frameworks; trauma-informed language wherever testimony is discussed.

## Definitions
**Trauma-informed language** — factual, dignified, never sensationalized; no real case examples or trigger imagery without explicit approval.

## Framework
Applies uniformly across `foundation/`, `methodology/`, `academy/`, `governance/`, `projects/`.

## Workflow
Every document is checked for register and trauma-informed compliance before being treated as final.

## Roles
Not specified — applies to all contributors.

## Inputs
Any draft document.

## Outputs
Documentation meeting the confirmed style bar.

## Governance
Style violations are corrected before publication, not after.

## AI Integration
AI-drafted content is held to the same style standard as human-drafted content.

## Examples
This documentation tree's own foundation/ and methodology/ documents are the reference implementation.
